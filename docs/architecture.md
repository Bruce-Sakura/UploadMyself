# 架构设计

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      Frontend (React + Three.js)            │

│  │ Skill UI │ │ Voice UI │ │ 2D Avatar│ │ 3D Avatar│      │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘ └────┬─────┘      │
└───────┼────────────┼────────────┼────────────┼──────────────┘
        │            │            │            │
        ▼            ▼            ▼            ▼
┌─────────────────────────────────────────────────────────────┐
│                    FastAPI Gateway                           │
│  /api/v1/skill  /api/v1/voice  /api/v1/avatar  /api/v1/... │
└──────────┬────────────┬────────────┬────────────────────────┘
           │            │            │
           ▼            ▼            ▼

│              Provider Layer (Local / Cloud)                  │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │

│  ML Models     │ │  Cloud APIs    │ │  Celery Workers    │
│  (GPU Inference)│ │  (OpenAI etc) │ │  (Async Tasks)     │
└────────────────┘ └────────────────┘ └────────────────────┘
```

## 核心设计原则

1. **Provider 抽象**：所有 ML 推理通过统一 Provider 接口，本地/云端可无缝切换
2. **异步优先**：重量级任务（训练、蒸馏、视频生成）走 Celery 异步队列
3. **模块独立**：四大引擎（Skill/Voice/2D/3D）互相独立，可单独部署
4. **模型注册**：通过 ModelRegistry 统一管理所有模型，支持热加载
